## Overview

Callouts are boxed sections used to highlight tips, warnings, success messages, and other important notes inside your Documentation.AI pages.

Use callouts when you need to:

- Draw attention to key information without breaking reading flow.

- Warn about breaking changes, security risks, or destructive actions.

- Provide tips and best practices alongside procedural steps.

- Confirm successful outcomes (for example, deployment complete, sync finished).

Callouts work alongside your regular [Headings and Text](/components/heading-and-text) and are often used with [Expandables](/components/expandables) when you want to hide long notes or advanced caveats behind a collapsible section.

Available callout kinds:

- `info`: Neutral information.

- `tip`: Helpful suggestions and best practices.

- `success`: Positive confirmations and results.

- `alert`: Warnings and cautions.

- `danger`: Critical errors and destructive actions.

## Using with Web Editor

In the Web Editor, you can insert callouts without writing any code.

1. Place your cursor where you want the callout.

2. Type <kbd>/</kbd> to open the block menu.

3. Search for **Callout** and choose the style you want (Info, Tip, Alert, Danger, or Success).

4. Start typing inside the callout block to add or edit content like a normal paragraph.

## Using with Code Editor

In MDX, use the `Callout` component with a `kind` attribute to control the base visual style. The `kind` value must be one of `info`, `tip`, `success`, `alert`, or `danger`.

You can also override the default icon with `icon`, override the background color with `color`, and control the initial expanded state with `collapsed`.

### Basic syntax

```jsx
<Callout kind="info">
  This is an informational callout.
</Callout>
```

### Info

Use for neutral notes or clarifications.

```jsx
<Callout kind="info">
  Configure your API key in the settings before making requests.
</Callout>
```

<Callout kind="info" collapsed="false">
  Configure your API key in the settings before making requests.
</Callout>

### Tip

Use for recommendations, shortcuts, or best practices.

```jsx
<Callout kind="tip">
  Use environment variables to store sensitive configuration like API keys.
</Callout>
```

<Callout kind="tip" collapsed="false">
  Use environment variables to store sensitive configuration like API keys.
</Callout>

### Alert

Use for warnings that require extra attention but are not strictly fatal.

```jsx
<Callout kind="alert">
  This action cannot be undone. Make sure you have a backup before proceeding.
</Callout>
```

<Callout kind="alert" collapsed="false">
  This action cannot be undone. Make sure you have a backup before proceeding.
</Callout>

### Danger

Use for destructive or high-risk actions.

```jsx
<Callout kind="danger">
  Deleting this workspace will permanently remove all documentation and cannot be recovered.
</Callout>
```

<Callout kind="danger" collapsed="false">
  Deleting this workspace will permanently remove all documentation and cannot be recovered.
</Callout>

### Success

Use for confirmations and positive outcomes.

```jsx
<Callout kind="success">
  Your documentation has been successfully deployed and is live.
</Callout>
```

<Callout kind="success" collapsed="false">
  Your documentation has been successfully deployed and is live.
</Callout>

### Custom callouts

Use `icon` and `color` when you want a callout to match a specific workflow, brand color, or UI pattern without changing the rest of your page layout.

```jsx
<Callout kind="info" icon="shield-check" color="#dbeafe">
  Review access rules before you publish private documentation.
</Callout>
```

<Callout kind="info" icon="shield-check" color="#dbeafe">
  Review access rules before you publish private documentation.
</Callout>

```jsx
<Callout kind="tip" icon="rocket" color="rgb(237, 233, 254)">
  Launch a preview branch before you merge major navigation changes.
</Callout>
```

<Callout kind="tip" icon="rocket" color="rgb(237, 233, 254)">
  Launch a preview branch before you merge major navigation changes.
</Callout>

The `icon` prop accepts any Lucide icon name in kebab-case, such as `shield-check`, `rocket`, or `circle-help`. The `color` prop accepts any valid CSS color value, including hex, `rgb(...)`, `hsl(...)`, and named colors.

### Attributes

<ParamField path="kind" param-type="string" required="false" deprecated="false">
  Base visual style for the callout. One of: `info` (default), `tip`, `success`, `alert`, `danger`.
</ParamField>

<ParamField path="icon" param-type="string" required="false" deprecated="false">
  Replaces the default kind icon. Use any Lucide icon name in kebab-case, such as `shield-check`, `rocket`, or `circle-help`.
</ParamField>

<ParamField path="color" param-type="string" required="false" deprecated="false">
  Replaces the kind-based background color. Accepts any valid CSS color value, including hex, `rgb(...)`, `hsl(...)`, and named colors.
</ParamField>

<ParamField path="collapsed" param-type="boolean" required="false" deprecated="false">
  Controls whether the callout starts collapsed. Use `"true"` to hide the content until the reader expands it, or `"false"` to keep it open.
</ParamField>

## Advanced options

Callouts support a collapsed state to help keep pages scannable when you have long explanations or optional details.

- `collapsed="false"`: The callout content is fully visible.

- `collapsed="true"`: The callout content is hidden until the reader expands it.

Example:

```jsx
<Callout kind="info" collapsed="true">
  This is an optional deep-dive explanation that starts collapsed.
</Callout>
```

When you need more structured or multi-paragraph content that should start hidden, combine callouts with [Expandables](/components/expandables) or use headings and body copy from [Headings and Text](/components/heading-and-text) inside the callout for clearer hierarchy.
