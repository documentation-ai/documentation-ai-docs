## Overview

The **ParamField** component lets you describe API parameters with:

- **Location badges:** Path, Query, Header, or Body

- **Type labels:** Such as string, integer, or boolean

- **Status tags:** Required and Deprecated

- **Rich descriptions:** Supporting any MDX content

Generated OpenAPI pages also use ParamField metadata for parameters. The live playground can show parameter examples from the connected OpenAPI spec, including named examples that readers can select from the input.

You can work with ParamField in two ways:

- **Editor UI:** Add and edit parameters visually with a form-based panel.

- **MDX:** Use the `<ParamField>` component directly in your content.

---

## Using with Web Editor

### Add a ParamField block

1. Place your cursor where you want the parameter description.
2. Type <kbd>/</kbd> to open the command menu and search for **Param Field** (or **Param**).
3. Select **Param Field** to insert the component.

A new parameter row appears with default badges and a description placeholder.

<Callout kind="info">

Prefer writing in code?

You can switch to **MDX view inside the Web Editor** to write or edit this component using the same syntax as the Code Editor. This is useful if you want full control while staying in the Web Editor.

</Callout>

To change the badges and tags:

1. Click the **three-dot menu** on the right side of the ParamField block.
2. In the **Parameter** settings panel, you can edit:

- **Location:** Choose **Path**, **Query**, **Body**, or **Header**.
- **Parameter Name:** Set the name shown next to the badge (for example, `id`, `limit`, `Authorization`).
- **Type:** Enter the data type (for example, `string`, `integer`, `boolean`).
- **Required:** Toggle on to mark the parameter as required.
- **Deprecated:** Toggle on to show the Deprecated badge.

Click **Save** to apply your changes.

### Edit the description

- Click in the **description area** under the badges (for example, *Parameter description…*).
- Add your explanation using any supported content:

  - Text and headings
  - Lists and tables
  - Code blocks
  - Callouts and other MDX components

## Using with Code Editor

### Basic syntax

Use the `<ParamField>` component to document API parameters with their location, type, and requirements.

```jsx
<ParamField path="id" param-type="string" required="true">
  Unique identifier for the resource.
</ParamField>
```

<ParamField path="id" param-type="string" required="true" deprecated="false">
  Unique identifier for the resource.
</ParamField>

### Hide location badge

Use `show-location="false"` to hide the location badge:

```jsx
<ParamField query="token" param-type="string" show-location="false" required="true">
  Authentication token without location badge.
</ParamField>
```

<ParamField query="token" param-type="string" required="true" deprecated="false">
  Authentication token without location badge.
</ParamField>

## Generated OpenAPI examples

Generated API docs use OpenAPI examples to make parameter inputs easier to test. Add examples to the OpenAPI parameter, not to the ParamField component itself.

```yaml
parameters:
  - name: status
    in: query
    schema:
      type: string
      enum: [active, invited, suspended]
    examples:
      activeUsers:
        summary: Active users
        value: active
      invitedUsers:
        summary: Pending invitations
        value: invited
```

When a connected OpenAPI parameter has one example, the playground can use it as the suggested input value. When the parameter has multiple named examples, the input includes an example picker that fills the selected value into the field.

<Callout kind="info" collapsed="false">
  Example pickers are suggestions, not validation rules. Use `enum` in your OpenAPI schema when readers must choose from a fixed set of allowed values.
</Callout>

## Advanced options

### Attributes

<ParamField path="path" param-type="string" required="false" deprecated="false">
  Parameter name for URL path parameters.
</ParamField>

<ParamField query="query" param-type="string" required="false" deprecated="false">
  Parameter name for query string parameters.
</ParamField>

<ParamField header="header" param-type="string" required="false" deprecated="false">
  Parameter name for HTTP headers.
</ParamField>

<ParamField body="body" param-type="string" required="false" deprecated="false">
  Parameter name for request body fields.
</ParamField>

<ParamField query="param-type" param-type="string" required="false" deprecated="false">
  Data type: `string`, `integer`, `boolean`, `object`, `array`, etc.
</ParamField>

<ParamField query="required" param-type="string" required="false" deprecated="false">
  Set to `"true"` for required parameters (default: `"false"`).
</ParamField>

<ParamField query="deprecated" param-type="string" required="false" deprecated="false">
  Set to `"true"` to mark as deprecated (default: `"false"`).
</ParamField>

<ParamField query="show-location" param-type="string" required="false" deprecated="false">
  Set to `"false"` to hide the location badge (default: `"true"`).
</ParamField>

<Callout kind="alert" collapsed="false">
  Use exactly one location attribute (`path`, `query`, `header`, or `body`) per ParamField.
</Callout>
