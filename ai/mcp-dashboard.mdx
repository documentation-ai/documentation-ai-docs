## Overview

The Dashboard MCP Server gives documentation owners and editors full programmatic access to their projects from any MCP-compatible AI client — Claude Desktop, Claude Code, Cursor, Windsurf, and others.

Unlike the [public MCP server](/ai/mcp-server) (which provides read-only access to published content), the Dashboard MCP Server authenticates you via your Documentation.AI account and provides:

- **Read and write access** to all your documentation projects
- **Multi-project support** — switch between organizations and documentation sites
- **Role-based permissions** matching your organization membership
- **Branch management** — create, merge, and delete branches
- **Live deployment** — push to the deployment branch to publish instantly

<Callout kind="info">
  The Dashboard MCP Server is available on **Standard plans and above**.
</Callout>

## Server URL

```
https://api.documentationai.app/mcp
```

## Authentication

The Dashboard MCP Server uses **OAuth 2.1** via your Documentation.AI account. When you connect from an MCP client, the authentication flow is handled automatically:

1. Your MCP client discovers the OAuth endpoints
2. A browser window opens for you to sign in to Documentation.AI
3. You approve access on the consent screen
4. The client receives tokens and connects

No API keys or manual token management required.

## Setup

### Claude Desktop

Add this to your Claude Desktop MCP configuration (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "documentationai": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://api.documentationai.app/mcp",
        "--timeout",
        "120000"
      ]
    }
  }
}
```

### Claude Code

Add the server from your terminal:

```bash
claude mcp add documentationai \
  --transport http \
  --url https://api.documentationai.app/mcp
```

Or add a `.mcp.json` file to your project root:

```json
{
  "mcpServers": {
    "documentationai": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://api.documentationai.app/mcp",
        "--timeout",
        "120000"
      ]
    }
  }
}
```

### Cursor

In Cursor settings, add a new MCP server:

- **Name:** `documentationai`
- **Type:** `command`
- **Command:** `npx mcp-remote https://api.documentationai.app/mcp --timeout 120000`

## First Steps After Connecting

Once authenticated, the MCP server requires you to select a project before using any documentation tools:

1. **`list_projects`** — See all organizations and documentation projects you have access to
2. **`select_project`** — Choose which project to work with
3. Then use any of the read or write tools below

You can switch projects at any time by calling `select_project` again.

## Tools Reference

### Project Selection (OAuth)

These tools are available immediately after authentication.

<Expandable title="list_projects">
  Lists all organizations and documentation projects you have access to, along with your role in each.

  **Parameters:** None

  **Returns:** A list of organizations, each containing their documentation projects and your role (admin, editor, or viewer).
</Expandable>

<Expandable title="select_project">
  Sets the active organization and documentation project for the current session.

  <ParamField body="organizationId" param-type="string" required="true" show-location="false">
    Organization ID from `list_projects`
  </ParamField>

  <ParamField body="documentationId" param-type="string" required="true" show-location="false">
    Documentation ID from `list_projects`
  </ParamField>
</Expandable>

### Read Tools

Available to all roles (viewer, editor, admin).

<Expandable title="get_site_config">
  Returns the complete `documentation.json` — the navigation tree, branding, SEO, and integrations.

  <ParamField body="branch" param-type="string" required="false" show-location="false">
    Branch name. Defaults to the deployment branch.
  </ParamField>
</Expandable>

<Expandable title="list_pages">
  Lists all files in the documentation repository with their paths and sizes.

  <ParamField body="branch" param-type="string" required="false" show-location="false">
    Branch name. Defaults to the deployment branch.
  </ParamField>
</Expandable>

<Expandable title="get_page">
  Returns the full content of a documentation page.

  <ParamField body="path" param-type="string" required="true" show-location="false">
    File path relative to repository root (e.g. `getting-started/introduction.mdx`)
  </ParamField>

  <ParamField body="branch" param-type="string" required="false" show-location="false">
    Branch name. Defaults to the deployment branch.
  </ParamField>
</Expandable>

<Expandable title="search_documentation">
  Performs a hybrid search combining semantic understanding and keyword matching.

  <ParamField body="query" param-type="string" required="true" show-location="false">
    Search query
  </ParamField>

  <ParamField body="limit" param-type="integer" required="false" show-location="false">
    Maximum results (1–50, default 10)
  </ParamField>
</Expandable>

### Write Tools

Require editor or admin role.

<Expandable title="update_documentation">
  Creates, updates, or deletes documentation files in a single Git commit. Supports up to 20 operations per request.

  <ParamField body="operations" param-type="array" required="true" show-location="false">
    Array of file operations. Each operation has `type` (create, update, or delete), `path`, and `content` (required for create/update).
  </ParamField>

  <ParamField body="commitMessage" param-type="string" required="true" show-location="false">
    Git commit message (max 500 characters)
  </ParamField>

  <ParamField body="branch" param-type="string" required="false" show-location="false">
    Target branch. Defaults to the deployment branch.
  </ParamField>
</Expandable>

<Expandable title="create_branch">
  Creates a new Git branch.

  <ParamField body="branchName" param-type="string" required="true" show-location="false">
    Name for the new branch
  </ParamField>

  <ParamField body="sourceBranch" param-type="string" required="false" show-location="false">
    Branch to create from. Defaults to the deployment branch.
  </ParamField>
</Expandable>

<Expandable title="list_branches">
  Lists all branches in the repository.

  **Parameters:** None
</Expandable>

<Expandable title="delete_branch">
  Deletes a branch. The deployment branch cannot be deleted.

  <ParamField body="branchName" param-type="string" required="true" show-location="false">
    Branch name to delete
  </ParamField>
</Expandable>

<Expandable title="merge_branches">
  Merges one branch into another. Merging into the deployment branch triggers a live deployment.

  <ParamField body="base" param-type="string" required="true" show-location="false">
    Target branch to merge into
  </ParamField>

  <ParamField body="head" param-type="string" required="true" show-location="false">
    Source branch to merge from
  </ParamField>

  <ParamField body="commitMessage" param-type="string" required="false" show-location="false">
    Merge commit message
  </ParamField>
</Expandable>

<Expandable title="list_pull_requests">
  Lists pull requests in the repository.

  <ParamField body="state" param-type="string" required="false" show-location="false">
    Filter by state: `open`, `closed`, or `all` (default: `open`)
  </ParamField>
</Expandable>

## Resources

The MCP server also exposes three resources that provide reference information:

| Resource | URI | Description |
| --- | --- | --- |
| Components Reference | `docs://components` | All available MDX components with props and usage examples |
| Config Schema | `docs://config-schema` | Complete `documentation.json` schema reference |
| Project Config | `docs://project/config` | The current project's `documentation.json` |

## Role-Based Access

Your access level is determined by your role in the selected organization:

| Role | Permissions |
| --- | --- |
| **Viewer** | Read tools only (get_site_config, list_pages, get_page, search_documentation) |
| **Editor** | Read + write tools (update_documentation, branches, merge, pull requests) |
| **Admin** | All tools |
