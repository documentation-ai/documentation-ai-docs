## Overview

Documentation.AI provides two MCP Servers for different use cases:

| | Public MCP | Dashboard MCP |
| --- | --- | --- |
| **Purpose** | Let external AI tools reference your published documentation | Give documentation owners full read/write access from AI coding tools |
| **Access** | Read-only | Read and write |
| **Authentication** | None (public) | OAuth via your Documentation.AI account |
| **URL** | `https://yourdomain.com/docs/_mcp` | `https://api.documentationai.app/mcp` |
| **Available on** | All plans | Standard and above |

## Public MCP Server

The public MCP server exposes your published documentation to external AI tools. It provides a single tool, `searchDocumentation`, which performs grounded search across your content using the same relevance ranking as your in-product AI Assistant.

This allows AI tools such as ChatGPT, Claude, Cursor, and other MCP-compatible clients to query and reference **your documentation**. Most MCP clients will automatically call `searchDocumentation` when the model needs additional context.

### What the public MCP server provides

- The `searchDocumentation` tool for querying your indexed documentation
- Unified search across pages, headings, API reference, and snippets
- The same ranking and grounding used by the in-app AI Assistant
- Read-only retrieval access (no content modification)

<Callout kind="alert">
  The public MCP server becomes available automatically when you publish the first version of your documentation.
</Callout>

### Where to access the public MCP server

You can find your MCP endpoint in the dashboard, under the **MCP Server** section in the sidebar.

Your MCP server URL typically follows one of these formats:

- `https://docs.yourdomain.com/_mcp`
- `https://yourdomain.com/docs/_mcp`

## Dashboard MCP Server

The Dashboard MCP Server authenticates you via your Documentation.AI account and gives you full access to your documentation projects from AI coding tools like Claude Desktop, Claude Code, and Cursor.

With the Dashboard MCP Server you can:

- Read pages, search content, and view site configuration
- Create, update, and delete documentation files
- Manage branches, merge changes, and trigger deployments
- Switch between multiple organizations and documentation projects
